    Test scenarios (from CONTEXT.md testing strategy):

    1. **FIFO queue drain**: Create 3 queue items with different createdAt. Verify push() is called in createdAt ASC order.
    2. **Successful sync removes item**: After push succeeds, verify markSynced(item.id) is called.
    3. **4xx error parks item**: Mock DioException with 400 status. Verify markParked() called, NOT retried.
    4. **5xx error retries with backoff**: Mock DioException with 500 status. Verify attemptCount incremented. Verify Future.delayed called with correct backoff duration.
    5. **Max retries resets count**: After 5 failed attempts, verify attemptCount reset to 0 (item stays as pending for next cycle).
    6. **Concurrent drain prevention**: Call drainQueue() while already syncing. Verify second call returns immediately (no duplicate processing).
    7. **pullDelta fetches and upserts**: Mock DioClient.get() returning SyncResponse JSON. Verify handler.applyPulled() called for each entity.
    8. **pullDelta updates cursor**: After successful pull, verify updateCursor() called with server_timestamp.
    9. **pullDelta with null cursor (first launch)**: Verify GET /api/v1/sync called without cursor param on first launch.
    10. **SyncStatus emissions**: Verify statusStream emits syncing/allSynced states during drain.

    Use ProviderContainer pattern from Phase 1 tests where applicable. Mocktail `when()` and `verify()` for assertions.

    Create `mobile/test/unit/core/sync/sync_queue_dao_test.dart`:

    5. **updateAttemptCount increments**: Insert item, update count to 3, verify count persisted.

    2. **Does not trigger on ConnectivityResult.none**: Mock stream emitting [ConnectivityResult.none]. Verify callback NOT called.
    3. **Dispose cancels subscription**: Call dispose(), verify stream subscription cancelled.
