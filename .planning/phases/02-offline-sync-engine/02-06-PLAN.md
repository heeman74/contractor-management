  <action>
Change the `cursor` query parameter type from `datetime | None` to `str | None` so that Pydantic does not reject empty strings at the validation layer. Then manually parse the cursor inside the handler body:

1. Change the parameter declaration:
   ```python
   cursor: str | None = Query(
       default=None,
       description="ISO8601 timestamp. Returns records changed after this time. "
                   "Omit or pass empty string for full download (first launch).",
   ),
   ```

2. Replace the single-line `since = cursor if cursor is not None else _EPOCH_START` with explicit parsing logic:
   ```python
   if cursor is None or cursor.strip() == "":
       since = _EPOCH_START
   else:
       try:
           since = datetime.fromisoformat(cursor)
       except ValueError:
           from fastapi import HTTPException
           raise HTTPException(
               status_code=422,
               detail=f"Invalid cursor format: '{cursor}'. Expected ISO8601 datetime.",
           )
   ```

3. Keep the `_EPOCH_START` constant and all downstream service calls unchanged.

This handles three cases: (a) cursor param absent -> None -> epoch, (b) cursor param empty string -> epoch, (c) cursor param with valid ISO8601 -> parsed datetime. Invalid non-empty strings still get a clear 422.
