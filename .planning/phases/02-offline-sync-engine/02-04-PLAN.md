    Create `mobile/lib/core/sync/workmanager_dispatcher.dart`:
    - Top-level function `callbackDispatcher()` with `@pragma('vm:entry-point')` annotation
    - Inside: `Workmanager().executeTask((task, inputData) async { ... })`
    - CRITICAL (Pitfall 1 from RESEARCH.md): Must call:
      ```dart
      WidgetsFlutterBinding.ensureInitialized();
      await setupServiceLocator();
      ```
      at the TOP of the task before accessing any getIt service — this is a separate Dart isolate with fresh memory
    - After re-init: `final syncEngine = getIt<SyncEngine>();`
    - Call `await syncEngine.drainQueue()` then `await syncEngine.pullDelta()`
    - Return `Future.value(true)` on success, handle errors gracefully (return true to avoid OS retry storm)
    - Import setupServiceLocator, getIt, SyncEngine

    Create `mobile/lib/core/sync/sync_status_provider.dart`:
    - `@riverpod` Stream provider `syncStatus`:
      - Get SyncEngine from getIt
      - Get ConnectivityService from getIt
      - Combine SyncEngine.statusStream with ConnectivityService.isOnlineStream
      - When offline: emit SyncStatus(offline, 0) regardless of queue count
      - When online: emit status from SyncEngine.statusStream (allSynced/pending/syncing)
    - Also create a simple `@riverpod` provider that watches the sync_queue pending count from SyncQueueDao.watchPendingCount() as a simpler alternative if the stream combination is complex
    - The provider must be a Stream provider so the UI reactively updates

    Update `mobile/lib/main.dart`:
    - Add WorkManager initialization after setupServiceLocator():
      ```dart
      Workmanager().initialize(callbackDispatcher, isInDebugMode: kDebugMode);
      Workmanager().registerPeriodicTask(
        'contractorhub-sync',
        'backgroundSync',
        frequency: const Duration(minutes: 15), // Android minimum — Pitfall 5
        constraints: Constraints(networkType: NetworkType.connected),
      );
      ```
    - Import callbackDispatcher from workmanager_dispatcher.dart
    - Import workmanager package
    - IMPORTANT: Do NOT add loading spinner on app open — show cached data immediately (user decision: no loading spinner)
    - Keep existing MaterialApp.router setup, ProviderScope, setupServiceLocator() call

    Follow existing patterns: @riverpod annotation (not StateNotifier), getIt for service access, ProviderScope for Riverpod.
