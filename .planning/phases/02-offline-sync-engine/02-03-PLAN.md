    Create `mobile/lib/core/sync/sync_engine.dart`:
    - SyncEngine class:
      - Constructor takes: AppDatabase, DioClient, SyncRegistry, ConnectivityService
      - Private `_isSyncing = false` lock flag (prevent concurrent drains)
      - Private `_syncStatusController = StreamController<SyncStatus>.broadcast()` for status updates
      - Static backoff delays: [1s, 2s, 4s, 8s, 16s]
      - Static maxAttempts = 5

    - `void initialize()`:
      - Start ConnectivityService listening, pass `_onConnectivityRestored` as callback
      - This is called once from setupServiceLocator or main

    - `Future<void> _onConnectivityRestored()`:
      - Call drainQueue() then pullDelta()

    - `Future<void> drainQueue()`:
      - If _isSyncing, return immediately (prevent re-entry)
      - Set _isSyncing = true, wrap in try/finally to reset
      - Get pending items from SyncQueueDao (FIFO order)
      - Emit SyncStatus(syncing, totalCount) at start
      - For each item:
        - Get handler from registry by item.entityType
        - Try: handler.push(item) -> dao.markSynced(item.id) (delete from queue)
        - Catch DioException:
          - If 4xx: dao.markParked(item.id, error: e.message) — park, don't retry (user decision)
          - If 5xx/timeout: increment attemptCount
            - If attemptCount >= maxAttempts: leave as pending, reset attemptCount to 0 (will retry on next connectivity cycle per user decision)
            - Else: await Future.delayed(backoffDelays[attemptCount - 1]), update attemptCount
        - Emit progress SyncStatus(syncing, current/total) after each item
      - After drain: emit SyncStatus based on remaining pending count

    - `Future<void> pullDelta()`:
      - Get current cursor from SyncCursorDao (null = first launch)
      - Call GET /api/v1/sync?cursor=<isoTimestamp> via DioClient (if cursor is null, omit param for full download)
      - Parse response JSON into entity lists
      - For each entity type in response:
        - Get handler from registry
        - For each entity: handler.applyPulled(entityData) — writes to Drift (upsert)
        - Handle deleted_at != null as soft-delete signal
      - Update cursor via SyncCursorDao.updateCursor(response.server_timestamp)
      - Drift streams auto-update UI when tables change — no manual notification needed

    - `Future<void> syncNow()`:
      - Public convenience method: drainQueue() then pullDelta()
      - Called by pull-to-refresh and foreground triggers

    - `Stream<SyncStatus> get statusStream` — exposes status for UI

    - `void dispose()`:
      - Cancel connectivity subscription
      - Close status stream controller

    SyncStatus class (can be in same file or separate):
    - enum SyncState { offline, allSynced, pending, syncing }
    - SyncStatus(SyncState state, int pendingCount, {int? syncingOf})
    - `String get subtitle` — switch expression per user decision:
      - offline -> 'Offline'
      - allSynced -> 'All synced'
      - pending -> '$pendingCount item(s) pending'
      - syncing -> 'Syncing $syncingOf of $pendingCount...'
