<objective>
Create the Drift local storage foundation for the offline sync engine: sync_queue outbox table, sync_cursor table, deleted_at soft-delete columns on existing entity tables, schema v2 migration, and DAOs for queue and cursor operations.

Purpose: INFRA-03 requires offline-first local storage. The sync_queue table is the transactional outbox that guarantees every local write is captured for later sync. The sync_cursor tracks delta pull progress. Soft delete enables tombstone propagation across devices.

Output: New Drift tables (sync_queue, sync_cursor), modified existing tables (deleted_at), updated AppDatabase (schema v2 + migration), SyncQueueDao, SyncCursorDao.
</objective>
