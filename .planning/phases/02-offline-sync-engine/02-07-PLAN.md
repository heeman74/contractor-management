Fix the two compounding bugs that prevent "Syncing 1 of 1..." from ever being visible in the UI.

**File 1: `mobile/lib/core/sync/sync_status_provider.dart`**

In the `syncStatus` async generator function, find the connectivity event handler (lines 51-59). The current code at lines 56-58 is:

```dart
} else {
  // Back online — resume showing engine status (or allSynced default).
  yield lastEngineStatus ?? const SyncStatus(SyncState.allSynced, 0);
}
```

Replace with:

```dart
} else {
  // Back online — do NOT yield here. The SyncEngine will emit its own
  // status (syncing -> allSynced) as it drains the queue via
  // _onConnectivityRestored(). Yielding a default here would race with
  // and overwrite the engine's syncing state, causing the UI to never
  // show "Syncing 1 of 1...".
  //
  // If the queue is empty, drainQueue() emits allSynced itself.
  // If the queue has items, drainQueue() emits syncing first.
  // Either way, the engine's own emission is the correct source of truth.
}
```

This removes the premature yield that was setting "All synced" before the engine had a chance to emit "Syncing...".

**File 2: `mobile/lib/core/sync/sync_engine.dart`**

In `drainQueue()`, after the initial syncing status emit (line 169) and before the item processing loop (line 171), add a single frame yield:

Current code:
```dart
_syncStatusController.add(SyncStatus(SyncState.syncing, total));

for (var i = 0; i < items.length; i++) {
```

Change to:
```dart
_syncStatusController.add(SyncStatus(SyncState.syncing, total));

// Yield to the event loop so Flutter can paint the "Syncing..." state
// before we start processing items. Without this, if the queue is small
// and the network is fast, drainQueue() can complete in a single frame
// and the UI only ever sees the final allSynced state.
await Future<void>.delayed(Duration.zero);

for (var i = 0; i < items.length; i++) {
```

This ensures at least one Flutter frame renders with the "Syncing N items..." status visible, regardless of how fast the actual sync completes.

Do NOT change any other logic in either file. The stream merging, offline handling, and engine event processing are all correct. These two surgical changes fix the race condition.
