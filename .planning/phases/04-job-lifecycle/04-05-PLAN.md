Create **mobile/lib/features/jobs/data/job_dao.dart** — `@DriftAccessor(tables: [Jobs, ClientProfiles, ClientProperties, JobRequests, SyncQueue])` class `JobDao`:

Methods (follow user_dao.dart transactional outbox pattern):
1. **watchJobsByCompany(companyId)** — Stream of Job rows WHERE companyId AND deletedAt IS NULL, ordered by createdAt DESC. Soft-delete filter required.
2. **watchJobsByContractor(contractorId)** — Stream WHERE contractorId AND deletedAt IS NULL. For contractor view.
3. **watchJobsByClient(clientId)** — Stream WHERE clientId AND deletedAt IS NULL. For CRM job history.
4. **insertJob(JobsCompanion)** — Transaction: insert into jobs table + insert SyncQueue entry with entityType='job', operation='create', payload=JSON-serialized job fields (manually build Map<String, dynamic> from Companion fields — do NOT use toColumns() which returns Map<String, Expression> per Phase 2 decision).
5. **updateJobStatus(jobId, newStatus, statusHistoryJson, newVersion)** — Transaction: update jobs row (status, statusHistory, version) + insert SyncQueue entry with operation='update' and payload including the transition details (new_status, version for optimistic locking).
6. **updateJob(jobId, companion)** — Transaction: update + SyncQueue entry.
7. **softDeleteJob(jobId)** — Transaction: set deletedAt + SyncQueue entry with operation='delete'.
8. **watchPendingRequestsByCompany(companyId)** — Stream of JobRequests WHERE status='pending' AND deletedAt IS NULL.
9. **insertJobRequest(JobRequestsCompanion)** — Transaction: insert + SyncQueue.
10. **watchClientProfiles(companyId)** — Stream of ClientProfiles WHERE companyId AND deletedAt IS NULL.
11. **upsertFromSync(table, companion)** — insertOnConflictUpdate for each table (used by sync handlers for pull).
