Create `backend/app/features/jobs/request_service.py` with `RequestService`:

Methods:
1. **submit_request(data, company_id, client_id=None, submitted_name=None, submitted_email=None, submitted_phone=None)** — Create JobRequest from form data. If client_id is None (web form submission), look up existing User by submitted_email within the company. If found, set client_id. If not found, create a new User with client role (reuses existing User creation pattern from Phase 1). Store photo file paths in the photos JSONB array (actual file writing is handled at the router level).

2. **list_pending_requests(company_id, offset, limit)** — All job_requests WHERE status='pending' AND deleted_at IS NULL, ordered by created_at ASC (oldest first for review queue). Eager-load client relationship.

3. **review_request(request_id, action, admin_user_id, decline_reason=None, decline_message=None)** — Handle admin review:
   - **accept**: Set request.status='accepted'. Create a new Job at Quote stage via JobService.create_job() with fields pre-filled from the request (description, trade_type, client_id, notes from request description). Set request.converted_job_id to the new job's ID. Return the created job.
   - **decline**: Set request.status='declined', store decline_reason (from preset list) and optional decline_message.
   - **info_requested**: Set request.status='info_requested'.

4. **get_request(request_id)** — Single request by ID with eager-loaded relationships.

5. **list_requests_for_client(client_id)** — All requests for a specific client (for client's in-app view of their submissions).

Create `backend/app/features/jobs/rating_service.py` with `RatingService`:

Methods:
1. **create_rating(job_id, rater_id, ratee_id, direction, stars, review_text, company_id)** — Validate:
   a. Job exists and status is 'complete' or 'invoiced'
   b. No existing rating for this job+direction (UNIQUE constraint)
   c. Rating window: find the most recent 'complete' entry in job.status_history JSONB. If more than 30 days ago, reject with 422.
   d. Create Rating record.
   e. Recalculate average_rating for the ratee's ClientProfile (if direction is admin_to_client) using AVG(stars) across all ratings WHERE ratee_id matches. Update the denormalized field via CrmRepository.

2. **update_rating(rating_id, stars, review_text, user_id)** — Update existing rating. Re-validate 30-day window. Recalculate average.

3. **get_ratings_for_job(job_id)** — Both directions for a single job.

4. **get_ratings_for_user(user_id)** — All ratings where ratee_id matches (for profile display).

Both services follow CLAUDE.md OOP rules. No db.commit(). Use specific exception types (not bare ValueError).
