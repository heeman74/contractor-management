Create **mobile/lib/features/jobs/presentation/screens/jobs_pipeline_screen.dart**:
- `class JobsPipelineScreen extends ConsumerWidget` — Replaces the placeholder jobs_screen.dart
- Toggle between KanbanBoard and filtered ListView via isKanbanView provider
- AppBar action: toggle button (ViewModule icon for kanban, ViewList icon for list)
- AppBar action: FAB or "+" button to navigate to job wizard
- List view: shows all jobs with filter chips at top (status, trade type, priority, contractor, client). Chips use ChoiceChip/FilterChip. Default sort: newest first.
- Batch operations: multi-select mode via long-press. Floating action button appears for bulk status transition (forward only). Use CheckboxListTile or Checkbox overlay on JobCards.
- Empty state: friendly message when no jobs exist, with prominent "Create your first job" button.
- Watches jobListNotifier provider for reactive data.

Create **mobile/lib/features/jobs/presentation/screens/job_wizard_screen.dart**:
- `class JobWizardScreen extends ConsumerStatefulWidget` — 4-step Stepper widget:
  - Step 1: Client selection (searchable dropdown from client profiles) + description TextField
  - Step 2: Address (pick from client's saved properties or enter new) + trade type dropdown
  - Step 3: Contractor selection (list of contractors, shows availability info if online) + scheduling fields (estimated duration, preferred date). Smart contractor suggestions: show top 3 sorted by trade match and client preference. Scheduling is optional at Quote stage.
  - Step 4: Review all entered data + priority selection + optional notes + Submit button
- On submit: calls jobDao.insertJob (offline-first, writes to Drift + sync queue)
- Offline detection: if offline, skip Step 3 scheduling and create at Quote stage (per locked decision). Show info message explaining scheduling requires connectivity.
- Form validation: description required (min 10 chars), trade type required, client optional but recommended.
- "Experienced admin fast path": all steps are tappable in the Stepper, allowing skip-ahead.

Create **mobile/lib/features/jobs/presentation/screens/job_detail_screen.dart**:
- `class JobDetailScreen extends ConsumerWidget` — Tabbed layout using TabBar + TabBarView:
  - **Details tab**: Description, trade type, priority, client info, contractor info, business fields (PO number, external ref, tags), notes. Edit button for admin to update fields.
  - **Schedule tab**: List of booking dates/times (from associated bookings — placeholder for Phase 5 calendar integration), contractor assignment, estimated duration, scheduled completion date.
  - **History tab**: Timeline view of status_history entries. Each entry shows: status badge, timestamp (formatted relative), user who made the change, reason (if backward transition). Uses ListView with timeline connector lines (simple custom paint or leading icon column).
- Status transition action: Bottom action bar with "Next Status" button (shows the allowed forward transition for the current user's role). Admin sees dropdown for all allowed transitions. Backward transition shows a TextFormField for required reason.
- Shows current status as a prominent colored chip at the top.

- Calls jobDao.updateJobStatus for transitions (offline-capable, no scheduling involved)
- Groups jobs: "Today", "Upcoming", "Completed" sections
- Empty state: "No jobs assigned to you"
