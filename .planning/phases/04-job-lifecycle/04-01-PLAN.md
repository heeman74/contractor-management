  <action>
Create migration 0008 using `op.execute(text(...))` for all DDL (consistent with migration 0007 pattern — Alembic autogenerate is unreliable for complex constraints).

Create 5 tables in dependency order:

1. **jobs** — id UUID PK, company_id FK, description TEXT NOT NULL, trade_type TEXT NOT NULL, status TEXT NOT NULL DEFAULT 'quote' with CHECK constraint (quote/scheduled/in_progress/complete/invoiced/cancelled), status_history JSONB NOT NULL DEFAULT '[]'::jsonb, priority TEXT NOT NULL DEFAULT 'medium' with CHECK (low/medium/high/urgent), client_id UUID FK users(id) nullable, contractor_id UUID FK users(id) nullable, purchase_order_number TEXT nullable, external_reference TEXT nullable, tags JSONB DEFAULT '[]'::jsonb, notes TEXT nullable, estimated_duration_minutes INTEGER nullable, scheduled_completion_date DATE nullable, search_vector TSVECTOR nullable, version INTEGER NOT NULL DEFAULT 1, created_at/updated_at/deleted_at timestamps. Enable RLS + FORCE RLS + tenant_isolation policy. Create indexes: GIN on search_vector, composite (company_id, status) WHERE deleted_at IS NULL, client_id WHERE deleted_at IS NULL, contractor_id WHERE deleted_at IS NULL. Create trigger for set_updated_at. Create function update_jobs_search_vector() + trigger for tsvector auto-update on description/notes INSERT or UPDATE.

2. **ALTER TABLE bookings ADD CONSTRAINT bookings_job_id_fkey FOREIGN KEY (job_id) REFERENCES jobs(id)** — this FK was explicitly deferred from migration 0007.

3. **client_profiles** — id UUID PK, company_id FK, user_id UUID NOT NULL UNIQUE FK users(id), billing_address TEXT nullable, tags JSONB DEFAULT '[]', admin_notes TEXT nullable, referral_source TEXT nullable, preferred_contractor_id UUID FK users(id) nullable, preferred_contact_method TEXT nullable, average_rating NUMERIC(3,2) nullable, version/created_at/updated_at/deleted_at. RLS + policy + set_updated_at trigger.

4. **client_properties** — id UUID PK, company_id FK, client_id UUID FK users(id), job_site_id UUID FK job_sites(id), nickname TEXT nullable, is_default BOOLEAN DEFAULT false, version/created_at/updated_at/deleted_at. RLS + policy + set_updated_at trigger.

5. **job_requests** — id UUID PK, company_id FK, client_id UUID FK users(id) nullable, description TEXT NOT NULL, trade_type TEXT nullable, urgency TEXT DEFAULT 'normal' CHECK (normal/urgent), preferred_date_start/end DATE nullable, budget_min/max NUMERIC(10,2) nullable, photos JSONB DEFAULT '[]', status TEXT DEFAULT 'pending' CHECK (pending/accepted/declined/info_requested), decline_reason TEXT nullable, decline_message TEXT nullable, converted_job_id UUID FK jobs(id) nullable, submitted_name/email/phone TEXT nullable, version/created_at/updated_at/deleted_at. RLS + policy + set_updated_at trigger.

6. **ratings** — id UUID PK, company_id FK, job_id UUID FK jobs(id), rater_id UUID FK users(id), ratee_id UUID FK users(id), direction TEXT NOT NULL CHECK (admin_to_client/client_to_company), stars INTEGER NOT NULL CHECK (1-5), review_text TEXT nullable, version/created_at/updated_at/deleted_at, UNIQUE (job_id, direction). RLS + policy + set_updated_at trigger.

Include a complete `downgrade()` that drops in reverse dependency order: ratings, job_requests, client_properties, client_profiles, bookings FK constraint, jobs, plus the search function.

Follow the exact pattern from 0007: all DDL via op.execute(text(...)), no Alembic ORM calls.
