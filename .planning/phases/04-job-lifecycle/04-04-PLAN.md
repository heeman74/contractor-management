**Job endpoints** (require auth via Depends(get_current_user)):
- `POST /jobs/` -- Create job. Accepts JobCreate body. Calls JobService.create_job. Returns 201 + JobResponse.
- `GET /jobs/` -- List jobs with query params: status, contractor_id, client_id, trade_type, priority, offset, limit. Returns list[JobResponse].
- `GET /jobs/search` -- Search jobs. Query param: q (search query) + filters. Returns list[JobResponse].
- `GET /jobs/{job_id}` -- Get single job. Returns JobResponse.
- `PATCH /jobs/{job_id}` -- Update job (non-lifecycle fields). Accepts JobUpdate body. Returns JobResponse.
- `PATCH /jobs/{job_id}/transition` -- Transition job status. Accepts JobTransitionRequest body (new_status, reason, version). Validates role from current_user. Catches InvalidTransitionError -> 422. Catches version mismatch -> 409. **CRITICAL: After calling JobService.transition_status(), if the new_status is 'scheduled', call SchedulingService to create bookings.** Specifically:
  - Inject SchedulingService via dependency injection (same pattern as JobService).
  - After successful transition to 'scheduled': if the job has estimated_duration_minutes <= 480 (single day), call `scheduling_service.book_slot(contractor_id=job.contractor_id, job_site_id=<from job>, start_time=<derive from scheduled_completion_date or job data>, duration_minutes=job.estimated_duration_minutes, job_id=job.id)`. If estimated_duration_minutes > 480 (multi-day), call `scheduling_service.book_multiday_job(contractor_id=job.contractor_id, job_site_id=<from job>, start_date=<derive from job data>, total_duration_minutes=job.estimated_duration_minutes, job_id=job.id)`.
  - Handle `BookingConflictError` from SchedulingService -> 409 with message "Scheduling conflict: the selected time slot is no longer available. Please choose a different time."
  - If booking creation fails, the transaction rolls back (no db.commit per CLAUDE.md), so the status transition is also reverted automatically.
  - Per CONTEXT.md locked decision: "Bookings are created when scheduling" and "Job wraps Booking(s)". This is where Phase 3's SchedulingService gets wired into Phase 4's job lifecycle.
- `DELETE /jobs/{job_id}` -- Soft delete (admin hard-removal). Returns 204.

**Contractor endpoints** (require auth):
- `GET /jobs/contractor/mine` -- Jobs assigned to current user (contractor_id = current_user.id). Returns list[JobResponse].

**Client CRM endpoints** (require auth, admin role):
- `GET /clients/` -- List client profiles with optional search. Returns list[ClientProfileResponse].
- `GET /clients/{user_id}` -- Get client profile with job history. Returns ClientProfileResponse + jobs.
- `POST /clients/{user_id}/profile` -- Create/update client profile. Returns ClientProfileResponse.
- `GET /clients/{user_id}/properties` -- List saved properties. Returns list[ClientPropertyResponse].
- `POST /clients/{user_id}/properties` -- Add saved property. Returns ClientPropertyResponse.
- `DELETE /clients/properties/{property_id}` -- Remove saved property. Returns 204.

**Job Request endpoints:**
- `GET /jobs/request/{company_id}` -- Render Jinja2 web form (NO auth required). Returns HTMLResponse.
- `POST /jobs/request/{company_id}` -- Submit job request from web form (NO auth, multipart/form-data). Accepts Form fields + File uploads (photos). Validates photo count (max 5) and content type (image/jpeg, image/png, image/heic) per Pitfall 7. Saves photos to `uploads/job_requests/{request_id}/` using aiofiles. Returns success HTML or 201 JSON.
- `POST /jobs/requests` -- Submit job request from in-app (auth required, JSON body). Returns 201 + JobRequestResponse.
- `GET /jobs/requests` -- List pending requests (admin). Returns list[JobRequestResponse].
- `GET /jobs/requests/{request_id}` -- Get single request. Returns JobRequestResponse.
- `POST /jobs/requests/{request_id}/review` -- Admin review action (accept/decline/info_requested). Returns JobRequestResponse (or JobResponse on accept).

**Rating endpoints** (require auth):
- `POST /jobs/{job_id}/ratings` -- Create rating. Returns RatingResponse.
- `PATCH /ratings/{rating_id}` -- Update rating. Returns RatingResponse.
- `GET /jobs/{job_id}/ratings` -- Get ratings for a job. Returns list[RatingResponse].

Exception handling at router level:
- InvalidTransitionError -> 422 with detail message
- Version mismatch -> 409 with "Job has been modified. Please refresh."
- BookingConflictError -> 409 with scheduling conflict message
- Rating window expired -> 422 with "Rating window has expired (30 days after completion)"

Register router in `backend/app/main.py`: `app.include_router(jobs_router, prefix="/api/v1")`. Also mount StaticFiles for uploads: `app.mount("/uploads", StaticFiles(directory="uploads"), name="uploads")`. Create the uploads directory if it doesn't exist (at app startup).

Router functions must be THIN per CLAUDE.md -- delegate all logic to service layer. Only schema validation, auth, exception mapping, and SchedulingService orchestration happen at the router.
