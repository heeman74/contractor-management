         - Up to 10 attachments enforced — disable buttons when limit reached, show snackbar "Maximum 10 attachments per note"
         - Optional one-line caption input per attachment (tap thumbnail to add/edit caption)
         - Save button: disabled when body is empty AND no attachments selected
       - On Save:
         1. Call NoteDao.insertNote(companyId, jobId, authorId, body) -> noteId
         2. For each attachment: call AttachmentDao.insertAttachment(companyId, noteId, type, localPath, thumbnailPath, caption, sortOrder)
         3. Close bottom sheet
         4. Show snackbar "Note saved" (attachments will upload on next sync)

    2. Photo compression helper (can be a static method or utility):
       - compressPhoto(String sourcePath) -> Future<(String compressedPath, String thumbnailPath)>

       - Generates thumbnail: maxWidth 200, quality 70
       - Saves both to app support directory under attachments/ subfolder
       - Returns paths to both files

    3. Create attachment_upload_service.dart:
       - AttachmentUploadService class: takes DioClient (from GetIt) and AttachmentDao
       - uploadPending() method:
         1. Get pending uploads from AttachmentDao.getPendingUploads()
         2. For each attachment:
            a. Set status to 'uploading' via AttachmentDao
            b. Build FormData with MultipartFile.fromFile(localPath), note_id, attachment_type, caption
            c. POST to /api/v1/files/upload
            d. On success: markUploaded(id, response remoteUrl)
            e. On DioException: incrementRetry(id). After 3 consecutive failures on same attachment, skip to next (park for next cycle).
         3. Upload on any connection (WiFi or cellular) per user decision
       - Retry logic: 3 retries with exponential backoff (5s, 15s, 45s delays between retries) per attachment
       - Progress tracking: expose uploadTotal (int) and uploadCompleted (int) as a Stream<(int total, int completed)> that emits after each upload completes/fails. The sync status provider will consume this stream.
