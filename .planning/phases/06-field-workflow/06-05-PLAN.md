  <action>
    1. Create timer_providers.dart:
       - timeEntryDaoProvider: Provider returning AppDatabase.timeEntryDao
       - TimerState class (not Freezed — simple class with equals): activeEntry (TimeEntryEntity?), elapsedSeconds (int), activeJobId (String?)
       - TimerNotifier extends AsyncNotifier<TimerState>:
         - build(): Query Drift for active session (clockedOutAt IS NULL for current user). If found, compute elapsedSeconds = now - clockedInAt, start Timer.periodic(1 second). Return TimerState with active entry.
         - clockIn(jobId, companyId):
           1. If already clocked in to same job, do nothing
           2. Call TimeEntryDao.clockIn(companyId, jobId, contractorId) — DAO handles auto-clock-out of previous job
           3. If job status was 'scheduled', call JobDao.updateJobStatus(jobId, 'in_progress') with status_history entry
           4. Start Timer.periodic(1 second) updating elapsedSeconds
           5. Update state with new active entry
         - clockOut():
           1. Stop Timer.periodic
           2. Call TimeEntryDao.clockOut(activeEntryId)
           3. Update state: activeEntry=null, elapsedSeconds=0, activeJobId=null
         - _tick(): increment elapsedSeconds, update state
         - dispose: cancel Timer.periodic
       - timerNotifierProvider: AsyncNotifierProvider<TimerNotifier, TimerState>
       - timeEntriesForJobProvider: StreamProvider.autoDispose.family<List<TimeEntryEntity>, String> — watches TimeEntryDao.watchEntriesForJob(jobId)

    2. Create timer_screen.dart:
       - TimerScreen ConsumerWidget: Takes jobId as parameter
       - GoRoute registered as push route: /timer/:jobId
       - Layout:
         - AppBar with job title/description
         - Large centered elapsed time display: HH:MM:SS format, large font (48sp+)
         - Job info card below timer (job description, client, address)
         - Large Clock In/Clock Out button (full width, primary color):
           - If no active session for this job: "Clock In" button (green)
           - If active session for this job: "Clock Out" button (red) + pulsing dot indicator
           - If active session for DIFFERENT job: "Clock In (will clock out of {other job})" button with warning text
         - Session history list below button:
           - Each session card: date, start time - end time, duration formatted as "Xh Ym"
           - Active session: shows "In progress" instead of end time, with live duration
         - Total time summary at bottom: sum of all completed sessions + current active session

    3. Create time_tracked_section.dart:
       - TimeTrackedSection ConsumerWidget: Takes jobId
       - Watches timeEntriesForJobProvider(jobId)
       - Displays in existing Schedule tab of job detail:
         - Section header: "Time Tracked"
         - Sessions grouped by date:
           - Date header (e.g., "March 11, 2026")
           - Under each date: rows showing contractor name, start - end, duration
           - Active sessions: "In progress" with live elapsed
         - Per-day subtotal
         - Overall total at bottom (bold)
         - If admin role: each session row has an edit icon that opens an adjust dialog:
           - Adjust dialog: editable start/end time fields, reason TextField (required)
           - On save: call TimeEntryDao.adjustEntry or similar (admin audit trail)
         - Empty state: "No time tracked yet"

    4. Add TimeTrackedSection to job_detail_screen.dart Schedule tab:
       - Insert TimeTrackedSection(jobId: job.id) at the bottom of the Schedule tab content

       - Add 'timer' to RouteNames constants
       - GoRoute(path: '/timer/:jobId', name: RouteNames.timer, builder: (context, state) => TimerScreen(jobId: state.pathParameters['jobId']!))
