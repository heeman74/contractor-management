Tables to create (all raw SQL via op.execute):

1. **contractor_schedule_locks** — Lightweight anchor for SELECT FOR UPDATE locking:
   - contractor_id UUID PRIMARY KEY REFERENCES users(id)
   - company_id UUID NOT NULL REFERENCES companies(id)
   - RLS policy: company_id = current_setting('app.current_company_id')::uuid

2. **contractor_weekly_schedule** — Per-contractor working hours template:
   - id UUID PRIMARY KEY DEFAULT gen_random_uuid()
   - company_id UUID NOT NULL REFERENCES companies(id)
   - contractor_id UUID NOT NULL REFERENCES users(id)
   - day_of_week INTEGER NOT NULL CHECK (day_of_week BETWEEN 0 AND 6) — 0=Monday
   - block_index INTEGER NOT NULL DEFAULT 0 — for multiple blocks per day
   - start_time TIME NOT NULL
   - end_time TIME NOT NULL
   - CHECK (end_time > start_time)
   - UNIQUE (contractor_id, day_of_week, block_index)
   - version INTEGER NOT NULL DEFAULT 1
   - created_at, updated_at, deleted_at standard timestamps
   - RLS policy on company_id
   - updated_at trigger (same pattern as migration 0002)

3. **contractor_date_overrides** — Date-specific schedule overrides:
   - id UUID PRIMARY KEY DEFAULT gen_random_uuid()
   - company_id UUID NOT NULL REFERENCES companies(id)
   - contractor_id UUID NOT NULL REFERENCES users(id)
   - override_date DATE NOT NULL
   - is_unavailable BOOLEAN NOT NULL DEFAULT false — true = entire day blocked
   - block_index INTEGER NOT NULL DEFAULT 0 — for multi-block overrides
   - start_time TIME — NULL when is_unavailable=true
   - end_time TIME — NULL when is_unavailable=true
   - CHECK ((is_unavailable = true AND start_time IS NULL AND end_time IS NULL) OR (is_unavailable = false AND start_time IS NOT NULL AND end_time IS NOT NULL AND end_time > start_time))
   - UNIQUE (contractor_id, override_date, block_index)
   - version, created_at, updated_at, deleted_at timestamps
   - RLS policy on company_id
   - updated_at trigger

4. **job_sites** — Geocoded job locations:
   - id UUID PRIMARY KEY DEFAULT gen_random_uuid()
   - company_id UUID NOT NULL REFERENCES companies(id)
   - address TEXT NOT NULL
   - latitude NUMERIC(9,6) NOT NULL
   - longitude NUMERIC(9,6) NOT NULL
   - name TEXT — optional site name
   - version, created_at, updated_at, deleted_at timestamps
   - RLS policy on company_id
   - updated_at trigger

5. **bookings** — Scheduled time blocks with GIST exclusion:
   - id UUID PRIMARY KEY DEFAULT gen_random_uuid()
   - company_id UUID NOT NULL REFERENCES companies(id)
   - contractor_id UUID NOT NULL REFERENCES users(id)
   - job_id UUID NOT NULL — FK will point to jobs table created in Phase 4; for now just UUID
   - job_site_id UUID REFERENCES job_sites(id) — nullable for now
   - time_range TSTZRANGE NOT NULL
   - day_index INTEGER — NULL for single-day; 0-based for multi-day
   - parent_booking_id UUID REFERENCES bookings(id) — links multi-day booking records
   - notes TEXT
   - version INTEGER NOT NULL DEFAULT 1
   - created_at, updated_at, deleted_at timestamps
   - EXCLUDE USING GIST (contractor_id WITH =, time_range WITH &&) WHERE (deleted_at IS NULL)
   - RLS policy on company_id
   - updated_at trigger
   - INDEX on contractor_id for availability queries
   - INDEX on job_id for job lookup

6. **travel_time_cache** — Cached ORS travel times:
   - id UUID PRIMARY KEY DEFAULT gen_random_uuid()
   - company_id UUID NOT NULL REFERENCES companies(id)
   - lat1 NUMERIC(9,6) NOT NULL
   - lng1 NUMERIC(9,6) NOT NULL
   - lat2 NUMERIC(9,6) NOT NULL
   - lng2 NUMERIC(9,6) NOT NULL
   - duration_seconds INTEGER NOT NULL
   - fetched_at TIMESTAMPTZ NOT NULL DEFAULT now()
   - UNIQUE (company_id, lat1, lng1, lat2, lng2)
   - INDEX on company_id

Additionally in the SAME migration:
- ALTER companies ADD COLUMN scheduling_config JSONB DEFAULT '{}'::jsonb
- ALTER users ADD COLUMN home_address TEXT, ADD COLUMN home_latitude NUMERIC(9,6), ADD COLUMN home_longitude NUMERIC(9,6), ADD COLUMN timezone TEXT DEFAULT 'UTC'

For all RLS policies, follow the established pattern from migration 0001:
- ENABLE ROW LEVEL SECURITY on each new table

For updated_at triggers, reuse the set_updated_at() function created in migration 0002 — just CREATE TRIGGER for each new table.
