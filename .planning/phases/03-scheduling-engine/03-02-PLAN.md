<objective>
Build the travel time and geocoding infrastructure with pluggable provider interfaces. The travel time system fetches driving duration from OpenRouteService, caches results in PostgreSQL with 30-day TTL and bidirectional key normalization, and applies company-configurable safety margins. The geocoding system converts addresses to lat/lng coordinates for travel time calculation and proximity sorting.

Purpose: SCHED-06 requires travel time awareness in scheduling. This plan creates the infrastructure that Plan 03's SchedulingService will consume to subtract travel buffers from free windows. The pluggable interface allows swapping to Google Maps later without touching scheduling logic.

Output: Travel time provider interface + ORS implementation + cache service, Geocoding provider interface + ORS geocoder
</objective>

```python
    ) -> int:
        """Return driving travel time in seconds."""
        ...
```

**backend/app/features/scheduling/travel/ors_provider.py** — ORS implementation:

- Class `OpenRouteServiceProvider(TravelTimeProvider)`
- Constructor takes `api_key: str` and `client: httpx.AsyncClient`
- Uses ORS Directions endpoint: `https://api.openrouteservice.org/v2/directions/driving-car`
- CRITICAL: ORS uses GeoJSON coordinate order — pass `f"{lng},{lat}"` (longitude first, NOT latitude first)
- Parse response: `data["features"][0]["properties"]["segments"][0]["duration"]` — returns seconds as float, convert to int
- Set timeout=10.0 on the request
- Handle httpx.HTTPStatusError and httpx.TimeoutException — re-raise as custom TravelTimeError
- Do NOT use openrouteservice-py — it's synchronous and blocks the event loop

**backend/app/features/scheduling/travel/cache.py** — Cache service:

- Class `TravelTimeCacheService`
- Constructor takes `db: AsyncSession` and `provider: TravelTimeProvider`
- Coordinate rounding: `round(value, 3)` for ~100m precision cache keys
- Bidirectional key normalization: sort the two coordinate pairs so (A,B) == (B,A) — reduces cache entries and API calls by half
- `async def get_travel_seconds(origin_lat, origin_lng, dest_lat, dest_lng, company_id) -> int`:
  1. Normalize and round coordinates
  2. Query TravelTimeCache table for matching entry
  3. If found and `fetched_at > now() - 30 days`: return duration_seconds
  4. If found but expired: store the expired value as fallback, then try API refresh
  5. If not found: call provider.get_travel_seconds()
  6. On provider success: upsert cache entry (INSERT ON CONFLICT UPDATE), return duration
  7. On provider failure: return expired fallback if available, else raise TravelTimeError

- Class `CachedTravelTimeProvider(TravelTimeProvider)` — wraps cache service as a provider interface so callers don't need to know about caching:
  - Constructor takes `cache_service: TravelTimeCacheService` and `company_id: uuid.UUID`
  - `get_travel_seconds()` delegates to cache_service.get_travel_seconds() with company_id

- Helper function `apply_safety_margin(seconds: int, margin_percent: float) -> int`:
  - Returns `int(seconds * (1 + margin_percent / 100))`
  - Used by scheduling service to add company-configured margin on top of raw travel time

Custom exceptions in provider.py or a shared exceptions file:
- `TravelTimeError(Exception)` — base for all travel time failures
- `TravelTimeAPIError(TravelTimeError)` — API returned non-200
- `TravelTimeUnavailableError(TravelTimeError)` — no cache fallback and API failed
