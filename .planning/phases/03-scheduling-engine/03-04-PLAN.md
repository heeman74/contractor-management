**backend/app/features/scheduling/router.py:**

Do NOT use CRUDRouter mixin for this domain — the scheduling endpoints are custom (not standard CRUD). Use a plain APIRouter with FastAPI dependency injection, following the pattern of thin router functions that delegate to SchedulingService.

All endpoints require `Depends(get_current_user)` per CLAUDE.md security rules.

**Availability endpoints:**

1. `POST /api/v1/scheduling/availability`:
   - Body: AvailabilityRequest (contractor_ids, trade_type, date, job_site_id)
   - Returns: list[AvailabilityResponse] with free_windows and blocked_intervals per contractor
   - Instantiate SchedulingService(db) with optional travel provider (construct CachedTravelTimeProvider if ORS API key is configured in settings)
   - Sort by distance if job_site_id provided

2. `GET /api/v1/scheduling/availability/{contractor_id}`:
   - Query params: date (required), job_site_id (optional)
   - Returns: single AvailabilityResponse for one contractor
   - Convenience endpoint for single-contractor lookup

**Booking endpoints:**

3. `POST /api/v1/scheduling/bookings`:
   - Body: BookingCreate (contractor_id, job_id, job_site_id, start, end, notes)
   - Returns: BookingResponse (201 Created)
   - Catch BookingConflictError -> 409 with ConflictDetail list in body
   - Catch OutsideWorkingHoursError -> 422
   - Catch BookingTooShortError -> 422

4. `POST /api/v1/scheduling/bookings/multi-day`:
   - Body: MultiDayBookingCreate (contractor_id, job_id, job_site_id, day_blocks, notes)
   - Returns: list[BookingResponse] (201 Created)
   - Catch BookingConflictError -> 409

5. `GET /api/v1/scheduling/bookings`:
   - Query params: contractor_id (optional), date_from, date_to (optional range filter)
   - Returns: list[BookingResponse]

6. `DELETE /api/v1/scheduling/bookings/{booking_id}`:
   - Soft-delete the booking
   - Returns: 204 No Content

7. `PATCH /api/v1/scheduling/bookings/{booking_id}/reschedule`:
   - Body: { start: datetime, end: datetime }
   - Returns: BookingResponse
   - Catch BookingConflictError -> 409

**Conflict check endpoint:**

8. `POST /api/v1/scheduling/conflicts`:
   - Body: { contractor_id, start, end }
   - Returns: list[ConflictDetail] (empty list = no conflicts)
   - Read-only pre-check, no lock acquisition

**Date suggestion endpoint:**

9. `POST /api/v1/scheduling/suggest-dates`:
   - Body: { contractor_id, num_days, preferred_start, duration_hours, within_days? }
   - Returns: list[DateSuggestion]

**Schedule management endpoints:**

10. `PUT /api/v1/scheduling/schedules/{contractor_id}/weekly/{day_of_week}`:
    - Body: { blocks: list[TimeBlock] }
    - Returns: list of created schedule entries (200)
    - Replaces all blocks for that contractor + day

11. `PUT /api/v1/scheduling/schedules/{contractor_id}/overrides/{date}`:
    - Body: DateOverrideCreate (is_unavailable, blocks)
    - Returns: list of created override entries (200)

12. `GET /api/v1/scheduling/schedules/{contractor_id}/weekly`:
    - Returns: full weekly schedule grouped by day_of_week

13. `GET /api/v1/scheduling/schedules/{contractor_id}/overrides`:
    - Query params: date_from, date_to
    - Returns: list of date overrides in range
