Create SchedulingRepository inheriting from TenantScopedRepository[Booking].

**Model assignment:** `model = Booking`

**Core query methods:**

1. `async def acquire_contractor_lock(self, contractor_id: UUID) -> None`:
   - SELECT FOR UPDATE on ContractorScheduleLock where contractor_id matches
   - If no lock row exists, create one (with company_id from tenant context), flush, then re-acquire with FOR UPDATE
   - Lock is released automatically when the enclosing transaction commits/rolls back
   - This serializes booking attempts for the SAME contractor while allowing parallel booking of DIFFERENT contractors

2. `async def get_bookings_in_range(self, contractor_id: UUID, range_start: datetime, range_end: datetime) -> list[Booking]`:
   - SELECT from bookings WHERE contractor_id = :cid AND time_range && tstzrange(:start, :end) AND deleted_at IS NULL
   - Use the `&&` overlap operator with TSTZRANGE
   - Use `selectinload` for job_site relationship if needed
   - Returns all bookings that overlap with the given range

3. `async def find_conflicts(self, contractor_id: UUID, utc_ranges: list[tuple[datetime, datetime]]) -> list[Booking]`:
   - For each range in utc_ranges, check overlap using `&&` operator
   - Combine with OR for all ranges in a single query
   - Returns conflicting bookings (if any)

4. `async def get_weekly_schedule(self, contractor_id: UUID) -> list[ContractorWeeklySchedule]`:
   - SELECT from contractor_weekly_schedule WHERE contractor_id = :cid AND deleted_at IS NULL
   - ORDER BY day_of_week, block_index

5. `async def get_date_overrides(self, contractor_id: UUID, start_date: date, end_date: date) -> list[ContractorDateOverride]`:
   - SELECT from contractor_date_overrides WHERE contractor_id = :cid AND override_date BETWEEN :start AND :end AND deleted_at IS NULL
   - ORDER BY override_date, block_index

6. `async def get_company_scheduling_config(self, company_id: UUID) -> SchedulingConfig`:
   - Get company by ID, parse scheduling_config JSONB into SchedulingConfig Pydantic model
   - Return default SchedulingConfig() if column is NULL or empty

7. `async def create_booking(self, booking: Booking) -> Booking`:
   - Use inherited create() method from BaseRepository
   - Handle IntegrityError from GIST constraint violation — catch and raise BookingConflictError with details

8. `async def get_contractor_with_location(self, contractor_id: UUID) -> User`:
   - SELECT user with home_latitude, home_longitude, timezone fields
   - For proximity sorting, also fetch the user's most recent booking's job_site for "last job location" fallback

9. `async def get_contractors_by_trade(self, trade_type: str, company_id: UUID) -> list[User]`:
   - Query users who have contractor role AND whose company has trade_type in trade_types array
   - OR: query user_roles WHERE role = 'contractor' AND company_id matches, then filter
   - For trade filtering, this can be refined in service layer

All methods use AsyncSession from self.db (inherited). No manual commit — get_db handles it.

Import IntegrityError from sqlalchemy.exc for GIST constraint violation handling.

<task type="auto">
  <name>Task 2: SchedulingService — availability, booking, multi-day, and date suggestion</name>
  <files>backend/app/features/scheduling/service.py</files>
  <action>
Create SchedulingService inheriting from TenantScopedService[Booking].

**Class setup:**
- `repository_class = SchedulingRepository`
- Constructor extends parent to accept optional `travel_provider: TravelTimeProvider | None = None` — when None, travel time is skipped (allows pure-logic unit testing without travel mocking)

**Private helper methods:**

1. `def _resolve_working_blocks(self, target_date: date, weekly_schedule: list[ContractorWeeklySchedule], date_overrides: list[ContractorDateOverride], contractor_tz: ZoneInfo, default_config: SchedulingConfig) -> list[tuple[datetime, datetime]]`:
   - Determine working hours for `target_date`:
     a. Check if `target_date` has a date override for this contractor
     b. If override exists AND is_unavailable=True: return empty list (day off)
     c. If override exists with custom blocks: use those blocks
     d. If no override: use contractor's weekly schedule for that day_of_week
     e. If no weekly schedule: fall back to company default_working_hours from SchedulingConfig
     f. If no company default: return empty list (not schedulable)
   - Convert each time block (start_time, end_time) to UTC datetimes for target_date using contractor's timezone
   - Use `datetime(year, month, day, hour, minute, tzinfo=contractor_tz).astimezone(ZoneInfo("UTC"))` — handles DST correctly
   - Return list of (UTC start, UTC end) tuples

2. `def _compute_free_windows(self, working_blocks: list[tuple[datetime, datetime]], blocked_intervals: list[tuple[datetime, datetime, str]], min_duration_minutes: int, buffer_minutes: int) -> tuple[list[FreeWindow], list[BlockedInterval]]`:
   - Implements the interval subtraction algorithm from RESEARCH.md Pattern 3
   - Expand each blocked interval by buffer_minutes on each side
   - Merge overlapping blocked intervals (sort by start, sweep)
   - Subtract from working blocks, keeping track of reasons for each blocked interval
   - Return only free windows >= min_duration_minutes
   - Also return list of BlockedInterval with reasons for each gap (feeds into Phase 5 calendar UI)
   - Reasons: "existing_job", "travel_buffer", "outside_working_hours"

3. `async def _get_travel_buffer(self, contractor_id: UUID, target_date: date, existing_bookings: list[Booking], config: SchedulingConfig) -> list[tuple[datetime, datetime, str]]`:
   - For each pair of consecutive bookings (by time), compute travel time between job sites
   - Use travel_provider.get_travel_seconds() if provider is available AND both bookings have job_site with lat/lng
   - Apply company safety margin: `apply_safety_margin(seconds, config.travel_margin_percent)`
   - Add buffer_minutes from config on top of travel time
   - Return list of blocked intervals representing travel + buffer time between consecutive bookings
   - If travel_provider is None or job site lacks coordinates: use config.default_travel_time_minutes as fallback
   - Handle TravelTimeUnavailableError: fall back to config.default_travel_time_minutes

4. `def _to_utc_range(self, day_block: DayBlock, contractor_tz: ZoneInfo) -> tuple[datetime, datetime]`:
   - Convert a DayBlock (date + start_time + end_time) to UTC datetime range
   - Use zoneinfo for DST-safe conversion

**Public methods:**

5. `async def get_available_slots(self, request: AvailabilityRequest) -> list[AvailabilityResponse]`:
   - Accept contractor_ids list OR trade_type filter (not both — if trade_type given, resolve to contractor IDs)
   - For each contractor:
     a. Get contractor timezone and home location
     b. Resolve working blocks for requested date
     c. Get existing bookings for that date range
     d. Compute travel buffers between existing bookings
     e. Compute free windows by subtracting bookings + travel buffers from working blocks
     f. Build AvailabilityResponse with free_windows and blocked_intervals
   - If request.job_site_id provided: compute distance from each contractor's last job site (or home base) for proximity sorting
   - Sort results by distance_km (nearest first)
   - Return list of AvailabilityResponse

6. `async def check_conflicts(self, contractor_id: UUID, start: datetime, end: datetime) -> list[ConflictDetail]`:
   - Query existing bookings that overlap with [start, end)
   - Return ConflictDetail for each conflict (booking_id, contractor_id, time range, job_id)
   - Does NOT acquire lock — this is a read-only pre-check

7. `async def book_slot(self, booking_data: BookingCreate) -> Booking`:
   - Acquire contractor lock (SELECT FOR UPDATE via repository)
   - Pre-check: call check_conflicts() to verify no overlap
   - If conflicts found: raise BookingConflictError with ConflictDetail list
   - Validate booking is within working hours (resolve working blocks, check containment)
   - Validate booking duration >= min_job_duration_minutes
   - Create Booking record with time_range = Range(start, end) (half-open interval)
   - GIST constraint provides safety net if lock somehow fails
   - Catch IntegrityError from GIST violation and convert to BookingConflictError
   - Return created Booking

8. `async def book_multiday_job(self, booking_data: MultiDayBookingCreate) -> list[Booking]`:
   - Acquire contractor lock
   - Convert all day_blocks to UTC ranges using contractor timezone
   - Pre-check ALL days for conflicts in one batch query
   - If ANY day has a conflict: raise BookingConflictError (all-or-nothing per user decision)
   - Validate each day is within working hours
   - Insert all booking records with day_index (0-based) and shared parent_booking_id (first booking's ID)
   - GIST constraint catches any races
   - Return list of all created Bookings

9. `async def suggest_dates(self, contractor_id: UUID, num_days: int, preferred_start: date, duration_hours: float, within_days: int = 30) -> list[DateSuggestion]`:
   - Search for available date combinations starting from preferred_start, within within_days window
   - For each candidate date, check if there's a free window >= duration_hours
   - First try consecutive date combinations, then non-consecutive
   - Return up to 5 DateSuggestion results, sorted by: consecutive first, then earliest start date
   - Per user decision: "Prefer consecutive dates when suggesting — suggest non-consecutive only if consecutive isn't available"

10. `async def reschedule_booking(self, booking_id: UUID, new_start: datetime, new_end: datetime) -> Booking`:
    - Soft-delete the existing booking
    - Book new slot (reuses book_slot logic)
    - If new booking fails (conflict), restore the old booking (remove soft-delete)
    - Travel time conflicts treated same as booking conflicts (hard rejection)

**Custom exceptions** (define at top of service.py or in a shared scheduling/exceptions.py):
- `BookingConflictError(Exception)`: conflicts: list[ConflictDetail]
- `OutsideWorkingHoursError(Exception)`: message about when working hours are
- `BookingTooShortError(Exception)`: requested duration vs minimum

**Schedule management methods** (CRUD for weekly schedule and date overrides):

11. `async def set_weekly_schedule(self, contractor_id: UUID, day_of_week: int, blocks: list[TimeBlock]) -> list[ContractorWeeklySchedule]`:
    - Delete existing blocks for contractor + day_of_week
    - Insert new blocks with sequential block_index
    - Create contractor_schedule_lock row if not exists

12. `async def set_date_override(self, contractor_id: UUID, override_date: date, is_unavailable: bool, blocks: list[TimeBlock] | None) -> list[ContractorDateOverride]`:
    - Delete existing overrides for contractor + override_date
    - If is_unavailable: insert single record with is_unavailable=True, no times
    - If not: insert blocks with sequential block_index

Follow all CLAUDE.md rules: lazy="raise" on relationships (already in models), no manual commit (get_db handles), specific exception types over ValueError.
