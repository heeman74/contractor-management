    Create the feature-first directory structure per RESEARCH.md architecture pattern:
    - `lib/core/database/` — Drift DB class and table definitions
    - `lib/core/network/` — Dio HTTP client
    - `lib/core/routing/` — (placeholder, wired in Plan 04)
    - `lib/core/di/` — get_it service locator
    - `lib/features/company/` — domain/, data/, presentation/
    - `lib/features/users/` — domain/, data/, presentation/
    - `lib/features/auth/` — presentation/providers/ (auth stub in Plan 04)
    - `lib/shared/widgets/` — reusable UI components
    - `lib/shared/models/` — shared enums (UserRole, TradeType)

    Create Drift table classes in separate files:
    - `tables/companies.dart`: Companies table with id (text, UUID clientDefault), name (text), address (text nullable), phone (text nullable), businessNumber (text nullable), logoUrl (text nullable), version (integer default 1), createdAt (dateTime), updatedAt (dateTime). Primary key: {id}.
    - `tables/users.dart`: Users table with id (text, UUID clientDefault), companyId (text, references Companies #id), email (text), firstName (text nullable), lastName (text nullable), phone (text nullable), version (integer default 1), createdAt (dateTime), updatedAt (dateTime). Primary key: {id}.
    - `tables/user_roles.dart`: UserRoles table with id (text, UUID clientDefault), userId (text, references Users #id), companyId (text, references Companies #id), role (text — 'admin' | 'contractor' | 'client'). Primary key: {id}.

    CRITICAL: Use `text().clientDefault(() => const Uuid().v4())()` for UUID columns. Override `Set<Column> get primaryKey => {id}` — NEVER use `customConstraint('PRIMARY KEY')` (see RESEARCH.md Pitfall 5).

    Create `app_database.dart` with `@DriftDatabase(tables: [Companies, Users, UserRoles])` annotation. Use `drift_flutter` package's `driftDatabase()` for connection opening. Schema version = 1. Use `MigrationStrategy(onCreate: (m) => m.createAll(), onUpgrade: stepByStep())`.

    Create `service_locator.dart`: `final getIt = GetIt.instance;` with `setupServiceLocator()` function that registers `AppDatabase()` as singleton.

    Create `dio_client.dart`: A configured Dio instance with base URL from environment/config (default `http://10.0.2.2:8000/api/v1` for Android emulator accessing host). Add interceptors for logging. Register in get_it.

    Run `dart run build_runner build --delete-conflicting-outputs` to generate all `.g.dart` files.
