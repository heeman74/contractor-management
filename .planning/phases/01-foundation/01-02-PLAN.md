    - `ContextVar[UUID | None]` for current_tenant_id
    - `get_current_tenant_id()` / `set_current_tenant_id()` helpers
    - `TenantMiddleware(BaseHTTPMiddleware)` extracting company_id from `X-Company-Id` header (stub for v2 JWT)

    Create domain-driven directory structure under `backend/app/features/`:

    - `users/`: models.py (SQLAlchemy User, UserRole models), schemas.py (Pydantic schemas), router.py (stub), service.py (stub)

    SQLAlchemy models for Company:
    - id: UUID primary key, server_default gen_random_uuid()
    - name: String NOT NULL
    - address, phone, business_number, logo_url: String nullable (full company profile per user decision)
    - version: Integer NOT NULL default 1

    - role: String with CHECK constraint ('admin', 'contractor', 'client')
    - created_at

    Create `migrations/env.py` configured for async SQLAlchemy: import all models from `app.features.*.models` so Alembic's autogenerate detects them. Use `run_async_migrations()` pattern with `connectable = create_async_engine()`.

    Create `migrations/versions/0001_initial.py` per RESEARCH.md Pattern 5:
    1. Extensions: `CREATE EXTENSION IF NOT EXISTS "uuid-ossp"` and `CREATE EXTENSION IF NOT EXISTS btree_gist` (idempotent — safe even though init.sql already ran them)
    2. Companies table (tenant root — NO RLS, no FORCE RLS)
    3. Users table (tenant-scoped) with company_id FK
    4. User_roles junction table (tenant-scoped) with user_id + company_id FKs, CHECK constraint on role
    5. Enable RLS: `ALTER TABLE users ENABLE ROW LEVEL SECURITY` + `FORCE ROW LEVEL SECURITY`
    6. Enable RLS: `ALTER TABLE user_roles ENABLE ROW LEVEL SECURITY` + `FORCE ROW LEVEL SECURITY`
    7. Tenant isolation policies using `current_setting('app.current_company_id', true)::uuid` — the `true` second arg is CRITICAL (see RESEARCH.md Pitfall 3)
    8. Indexes: idx_users_company_id, idx_user_roles_company_id, idx_user_roles_user_id

    CRITICAL constraints from RESEARCH.md:
    - NEVER apply `FORCE ROW LEVEL SECURITY` on the companies table (it IS the tenant root)
    - Always use `current_setting('app.current_company_id', true)` — the `true` prevents errors when no tenant context is set (e.g., Alembic migrations, superuser sessions)
    - version column on ALL entities from day one

    Include proper `downgrade()` that drops policies, then tables in reverse order.

    Create `.pre-commit-config.yaml` per RESEARCH.md Pattern 6: ruff-check with --fix and ruff-format hooks.
