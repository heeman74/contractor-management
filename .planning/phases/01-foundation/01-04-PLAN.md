  <action>
    Create `auth_state.dart` with Freezed AuthState:
    ```dart
    @freezed
    sealed class AuthState with _$AuthState {
      const factory AuthState.loading() = AuthLoading;
      const factory AuthState.unauthenticated() = AuthUnauthenticated;
      const factory AuthState.authenticated({
        required String userId,
        required String companyId,
        required Set<UserRole> roles,
      }) = AuthAuthenticated;
    }
    ```
    Uses `UserRole` enum from `shared/models/user_role.dart`. `Set<UserRole>` supports multi-role per user decision.

    Create `auth_provider.dart` with `@riverpod` AuthNotifier:
    - `build()` returns `AuthState.loading()` initially
    - `setMockUser({required String userId, required String companyId, required Set<UserRole> roles})` — Phase 1 testing stub that sets authenticated state with specified roles. This will be replaced by real JWT auth in v2.
    - `logout()` — sets state to unauthenticated

    Create `route_names.dart` with const route path strings to avoid magic strings:
    - `/splash`, `/onboarding`, `/unauthorized`
    - `/home`, `/jobs`, `/schedule`, `/profile`
    - `/admin/team`, `/admin/clients`
    - `/contractor/availability`
    - `/client/portal`

    Create `app_router.dart` per RESEARCH.md Pattern 3:
    - Create `routerProvider` as a `Provider<GoRouter>` that reads `authNotifierProvider`
    - Use `ValueNotifier<AuthState>` bridge pattern: create ValueNotifier with initial auth state, `ref.listen(authNotifierProvider, ...)` to keep it in sync. Pass as `refreshListenable` to GoRouter. CRITICAL: Do NOT `watch` authNotifierProvider — that rebuilds the router (Pitfall 4). Use `listen` + ValueNotifier bridge instead.
    - Redirect logic:
      - AuthLoading state -> redirect to /splash
      - AuthUnauthenticated state -> redirect to /onboarding
      - AuthAuthenticated state:
        - Routes starting with `/admin` require `UserRole.admin` in roles set
        - Routes starting with `/contractor` require `UserRole.contractor`
        - Routes starting with `/client` require `UserRole.client`
        - Unauthorized access redirects to `/unauthorized`
    - Route structure:
      - GoRoute: /splash -> SplashScreen
      - GoRoute: /onboarding -> OnboardingScreen
      - GoRoute: /unauthorized -> UnauthorizedScreen
      - ShellRoute (builder: AppShell):
        - GoRoute: /home -> HomeScreen
        - GoRoute: /jobs -> JobsScreen
        - GoRoute: /schedule -> ScheduleScreen
        - GoRoute: /profile -> ProfileScreen
        - GoRoute: /admin/team -> TeamManagementScreen (placeholder)
        - GoRoute: /admin/clients -> ClientManagementScreen (placeholder)
        - GoRoute: /contractor/availability -> AvailabilityScreen (placeholder)
        - GoRoute: /client/portal -> ClientPortalScreen (placeholder)

    Run build_runner to generate Riverpod and Freezed code.
  </action>
