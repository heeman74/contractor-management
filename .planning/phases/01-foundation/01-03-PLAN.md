    Create shared enums:

    - `trade_type.dart`: `enum TradeType { builder, electrician, plumber, hvac, painter, carpenter, roofer, landscaper, general }` — supports multiple trades per contractor per user decision.

    Create Freezed domain entities:
    - `company_entity.dart`: `@freezed class CompanyEntity` with: id (String), name (String), address (String?), phone (String?), tradeTypes (List<TradeType>?), logoUrl (String?), businessNumber (String?), version (int), createdAt (DateTime), updatedAt (DateTime). Include `fromJson`/`toJson` via json_serializable.
    - `user_entity.dart`: `@freezed class UserEntity` with: id (String), companyId (String), email (String), firstName (String?), lastName (String?), phone (String?), version (int), createdAt (DateTime), updatedAt (DateTime). Include `fromJson`/`toJson`.
    - `user_role_entity.dart`: `@freezed class UserRoleEntity` with: id (String), userId (String), companyId (String), role (UserRole), createdAt (DateTime). Include `fromJson`/`toJson`.

    Create Drift DAOs:
    - `company_dao.dart`: `@DriftAccessor(tables: [Companies]) class CompanyDao extends DatabaseAccessor<AppDatabase>`. Methods:
      - `Stream<List<Company>> watchAllCompanies()` — reactive stream of all companies
      - `Future<Company> getCompanyById(String id)` — single company lookup
      - `Future<int> insertCompany(CompaniesCompanion entry)` — insert
      - `Future<bool> updateCompany(CompaniesCompanion entry)` — update
      - `Future<int> deleteCompany(String id)` — delete
    - `user_dao.dart`: `@DriftAccessor(tables: [Users, UserRoles]) class UserDao extends DatabaseAccessor<AppDatabase>`. Methods:
      - `Stream<List<User>> watchUsersByCompany(String companyId)` — reactive stream filtered by company (local tenant filter)
      - `Future<User> getUserById(String id)`
      - `Stream<List<UserRole>> watchRolesForUser(String userId)` — reactive stream of roles
      - `Future<void> insertUser(UsersCompanion entry)`
      - `Future<void> assignRole(UserRolesCompanion entry)`

    CRITICAL: All read methods that power UI must return `Stream<>` (Drift watch queries), NOT `Future<>`. This is the offline-first pattern — UI streams from local DB, never awaits HTTP. See RESEARCH.md Pitfall 7.

    Update `app_database.dart` to include DAOs: `@DriftDatabase(tables: [Companies, Users, UserRoles], daos: [CompanyDao, UserDao])`.

    Create Riverpod providers:
    - `company_providers.dart`: `@riverpod Stream<List<CompanyEntity>> companies(Ref ref)` that gets CompanyDao from get_it and calls `watchAllCompanies()`, mapping Drift row objects to Freezed entities.
    - `user_providers.dart`: `@riverpod Stream<List<UserEntity>> companyUsers(Ref ref, String companyId)` that streams users for a given company.

    Register DAOs in get_it service_locator.dart (or access via `getIt<AppDatabase>().companyDao`).

  <action>
    Create Pydantic schemas for companies:
    - `CompanyCreate`: name (str, min_length=1), address (str | None), phone (str | None), trade_types (list[str] | None), logo_url (str | None), business_number (str | None)
    - `CompanyResponse`: all fields from create + id (UUID), version (int), created_at (datetime), updated_at (datetime). Use `model_config = ConfigDict(from_attributes=True)` for SQLAlchemy ORM mode.
    - `CompanyUpdate`: all fields optional (partial update)

    Create Pydantic schemas for users:
    - `UserCreate`: email (str, email format), first_name (str | None), last_name (str | None), phone (str | None)
    - `UserResponse`: all fields + id (UUID), company_id (UUID), version (int), created_at, updated_at, roles (list[str])
    - `RoleAssignment`: user_id (UUID), role (Literal['admin', 'contractor', 'client'])

    Create CompanyService class:
    - `create_company(db: AsyncSession, data: CompanyCreate) -> Company` — creates company (no tenant filter — company IS the tenant root)
    - `get_company(db: AsyncSession, company_id: UUID) -> Company | None`
    - `update_company(db: AsyncSession, company_id: UUID, data: CompanyUpdate) -> Company`

    Create UserService class:
    - `create_user(db: AsyncSession, data: UserCreate) -> User` — creates user scoped to current tenant (company_id from ContextVar, NOT from request body — see RESEARCH.md anti-pattern)
    - `list_users(db: AsyncSession) -> list[User]` — RLS automatically filters to current tenant
    - `assign_role(db: AsyncSession, data: RoleAssignment) -> UserRole`
    - `get_user_roles(db: AsyncSession, user_id: UUID) -> list[UserRole]`

    CRITICAL: company_id for user creation MUST come from `get_current_tenant_id()` ContextVar (set by TenantMiddleware), NEVER from the request body. Accepting company_id from client input is a tenant isolation bypass vulnerability.

    Create company router (`/api/v1/companies`):
    - `POST /` — create company
    - `GET /{company_id}` — get company by ID

    Create user router (`/api/v1/users`):
    - `POST /` — create user (company_id from middleware)
    - `GET /` — list users (RLS-filtered)
    - `POST /{user_id}/roles` — assign role to user
    - `GET /{user_id}/roles` — get user's roles

    Update `main.py` to include both routers with `/api/v1` prefix.
  </action>
