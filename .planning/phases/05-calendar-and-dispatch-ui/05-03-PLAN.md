    **Create `mobile/lib/features/schedule/presentation/widgets/unscheduled_jobs_drawer.dart`:**
    A collapsible side drawer showing jobs without bookings:
    - Slides in from the right edge of the screen. Toggle button (hamburger/filter icon) on the schedule screen header.
    - Contents: filterable list of unscheduled jobs (status = 'quote' or 'scheduled' with no booking). Filters: status chips, trade type dropdown, priority dropdown, client search text field.
    - Each job card is wrapped in `LongPressDraggable<BookingDragData>`:
      - `data: BookingDragData(jobId: job.id, durationMinutes: job.estimatedDurationMinutes ?? 60)`
      - `feedback`: Material-elevated mini card (width: laneWidth, height: durationToPixels) showing job description + client name
      - `childWhenDragging`: same card at 0.3 opacity
      - `hapticFeedbackOnStart: true` for tactile confirmation
    - Data source: watch jobs from JobDao where no matching booking exists in BookingDao for the current date. This is a derived provider combining jobs stream and bookings stream.
    - Empty state: "All jobs scheduled" message when no unscheduled jobs remain.

    **Update `mobile/lib/features/schedule/presentation/widgets/contractor_lane.dart`:**
    Add DragTarget grid overlay on each contractor lane:
    - The lane's Stack now includes a column of thin DragTarget<BookingDragData> strips behind the booking cards. Each strip represents a 15-minute time slot.
    - `onWillAcceptWithDetails`: check local Drift bookings for time overlap. Query: does any existing booking on this contractor overlap with [slotStart, slotStart + dragData.durationMinutes]? Return true if no conflict, false if conflict.
    - Visual feedback in builder: `candidateData.isNotEmpty` -> green highlight on the strip (Colors.green.withOpacity(0.2)); `rejectedData.isNotEmpty` -> red highlight (Colors.red.withOpacity(0.2)); non-working hours remain grayed (not green).
    - `onAcceptWithDetails`: call calendar provider method to create booking (see below). The drag data includes jobId and durationMinutes. The drop position gives contractorId and slotStartTime.

    **Update `mobile/lib/features/schedule/presentation/providers/calendar_providers.dart`:**
    Add an AsyncNotifier or methods accessible to the UI for booking operations:
    - `bookSlot(String contractorId, String jobId, DateTime slotStart, int durationMinutes)`:
      1. Insert booking into Drift via BookingDao.insertBooking() (offline-first — no HTTP)
      2. If job status is 'quote', update job status to 'scheduled' via JobDao (auto-transition per CONTEXT.md)
      3. Push the last operation to an undo stack (StateProvider<List<UndoAction>>)
      4. Show undo snackbar (via a callback or global key — the UI layer handles SnackBar display)
    - `reassignBooking(String bookingId, String newContractorId, DateTime newStart)`:
      1. Update booking in Drift via BookingDao.updateBookingTime (change contractorId + time)
      2. Push to undo stack
    - `undoLastBooking()`:
      1. Pop from undo stack
      2. If it was a CREATE, soft-delete the booking
      3. If it was a REASSIGN, restore original contractorId + time
      4. If it was a RESIZE, restore original start/end times
    - `UndoAction` class: { type: create|reassign|resize|delete, bookingId, previousState }
    - Undo stack: StateProvider<List<UndoAction>> with max depth 10

    **Update `mobile/lib/features/schedule/presentation/screens/schedule_screen.dart`:**
    - Add toggle button in header for unscheduled jobs drawer (icon: list/queue icon, toggles drawer visibility)
    - Integrate UnscheduledJobsDrawer as a positioned overlay or AnimatedContainer sliding in from right
    - When drawer is open, calendar view narrows or drawer overlays calendar with semi-transparent background
    - Show undo SnackBar after every booking operation with `duration: const Duration(seconds: 5)` (REQUIRED — Flutter 3.29+ breaking change: SnackBar with action does NOT auto-dismiss). SnackBarAction label "Undo" calls undoLastBooking().

    IMPORTANT: Conflict check during drag is LOCAL ONLY (read from Drift bookings stream) — no HTTP call during drag. This ensures instant feedback even offline.
    IMPORTANT: Booking creation writes to Drift + sync queue (offline-first pattern). The sync engine pushes to backend asynchronously.
    IMPORTANT: DragTarget strips must NOT be built as 1440 individual widgets. Use a ListView.builder or SliverList with 15-minute slots only for the visible working-hours range. Total widgets: ~48-64 per lane (8-16 working hours * 4 slots/hour), not 96 (full 24h).

    - Add two GestureDetector strips at the top and bottom edges of each BookingCard (8px tall, cursor: resize handle).
    - Top edge drag: adjusts timeRangeStart (drag up = earlier start, drag down = later start). Snap to 15-minute increments.

    - On release: call a `resizeBooking(bookingId, newStart, newEnd, currentVersion)` method in the provider.
    - Conflict check: if resized range overlaps another booking, snap back to original size and show conflict snackbar.
    - Minimum booking duration: 15 minutes (prevent resizing to zero).

    **Multi-day wizard (create multi_day_wizard_dialog.dart):**

      4. "Suggest dates" button that calls the backend suggest-dates endpoint (via Dio) and pre-fills recommended dates/times
      5. Cancel button: undoes the first-day booking (calls undoLastBooking) and closes
      6. Confirm button: creates all additional day bookings in Drift via BookingDao with dayIndex and parentBookingId linking to the first booking
    - Each additional day booking shares the same jobId and parentBookingId (first booking's id).
