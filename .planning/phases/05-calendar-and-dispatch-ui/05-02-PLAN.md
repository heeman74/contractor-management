  <action>
    **Add patterns_canvas dependency:**
    Run `cd mobile && flutter pub add patterns_canvas` (or manually add `patterns_canvas: ^0.5.0` to pubspec.yaml dependencies).

    **Create `mobile/lib/features/schedule/domain/overdue_service.dart`:**
    Pure Dart class with static methods (no dependencies):
    - `enum OverdueSeverity { none, warning, critical }`
    - `static OverdueSeverity computeSeverity(DateTime? scheduledCompletionDate)` — returns none if null or not past, warning if 1-3 days overdue, critical if 4+ days. Compare using `DateTime.now().difference(scheduledCompletionDate).inDays`.
    - `static bool isOverdue(String status, DateTime? scheduledCompletionDate)` — returns true if status is 'scheduled' or 'in_progress' AND today > scheduledCompletionDate.

    **Create `mobile/lib/features/schedule/presentation/providers/calendar_providers.dart`:**
    State providers (import from `package:flutter_riverpod/legacy.dart` for StateProvider):
    - `calendarDateProvider` — StateProvider<DateTime> defaulting to DateTime.now() (selected date for day view)
    - `calendarViewModeProvider` — StateProvider<CalendarViewMode> defaulting to CalendarViewMode.day (enum: day, week, month)
    - `contractorPageIndexProvider` — StateProvider<int> defaulting to 0 (current page of paginated contractor lanes, 5 per page)
    - `showCompletedJobsProvider` — StateProvider<bool> defaulting to false (toggle to include Complete/Invoiced/Cancelled)
    - `tradeTypeFilterProvider` — StateProvider<String?> defaulting to null (filter contractors by trade type)

    Data providers (using Riverpod annotation pattern):
    - `bookingsForDateProvider(DateTime date)` — AsyncNotifier or StreamProvider that watches BookingDao.watchBookingsByCompanyAndDateRange for the selected date. Get companyId from the current user's auth state.
    - `contractorsProvider` — provider that returns list of active contractors from UserDao (filter by contractor role, company scoped).
    - `filteredContractorsProvider` — derived provider that applies tradeTypeFilter to contractorsProvider results, then paginates (5 per page using contractorPageIndexProvider).

    **Create `mobile/lib/features/schedule/presentation/providers/overdue_providers.dart`:**
    - `overdueJobsProvider` — derived provider filtering all local jobs (from existing jobsByCompanyProvider or JobDao stream) where `OverdueService.isOverdue(status, scheduledCompletionDate)` is true.
    - `overdueJobCountProvider` — derived provider returning `.length` of overdueJobsProvider.

    Status color map (can be in calendar_providers.dart or separate):
    ```dart
    const statusColorMap = {
      'quote': Colors.grey,
      'scheduled': Colors.blue,
      'in_progress': Colors.orange,
      'complete': Colors.green,
      'invoiced': Colors.purple,
      'cancelled': Colors.red,
    };
    ```

    **Create `mobile/lib/features/schedule/presentation/widgets/calendar_grid_painter.dart`:**
    CustomPainter that draws the time-axis background grid for a single contractor lane:
    - Draws horizontal lines at each hour mark (0-23) using `pixelsPerMinute` scale factor
    - Paints non-working-hour regions in grey (receives list of blocked intervals with reason "outside_working_hours" or "time_off")
    - Paints "day off" / "holiday" labels centered in blocked regions with reason "time_off"
    - Draws a red horizontal "now" line at the current time position (receives currentTime parameter)
    - `shouldRepaint` returns true only when data changes (compare blocked intervals list and currentTime minute)

    **Create `mobile/lib/features/schedule/presentation/widgets/booking_card.dart`:**
    Stateless widget displaying a single booking on the calendar:
    - Sized based on booking duration (timeRangeEnd - timeRangeStart) * pixelsPerMinute
    - Background color from statusColorMap[job.status] with 0.15 opacity fill + solid left border (4px)
    - Content: job description (truncated, max 2 lines) + client name (1 line) in small text
    - Overdue indicator: if OverdueService.computeSeverity returns warning, add yellow/orange border; if critical, red border + small warning Icon
    - Delay indicator: if job's status_history contains a "delay" entry, show small clock icon badge
    - When showCompletedJobs is false AND job status is complete/invoiced/cancelled, render with 0.4 opacity (dimmed)
    - Active (non-completed) jobs only shown by default

    **Create `mobile/lib/features/schedule/presentation/widgets/travel_time_block.dart`:**
    Widget rendering a hatched/striped block for travel time between consecutive bookings:
    - Takes height (from travel buffer duration * pixelsPerMinute) and width (lane width)
    - Uses CustomPaint with patterns_canvas DiagonalStripesLight(bgColor: Colors.grey.shade200, fgColor: Colors.grey.shade400)
    - Small centered label "Travel" in grey text if block is tall enough (>30px)
    - No interaction — purely visual

    **Create `mobile/lib/features/schedule/presentation/widgets/contractor_lane.dart`:**
    Widget rendering one contractor's day schedule as a vertical time column:
    - Takes contractorId, date, bookings list, blocked intervals, pixelsPerMinute
    - Uses a Stack: CustomPaint(painter: CalendarGridPainter) as background, then absolutely positioned BookingCard widgets and TravelTimeBlock widgets
    - BookingCard positioned at `(timeRangeStart - dayStart).inMinutes * pixelsPerMinute` from top
    - TravelTimeBlock positioned between consecutive bookings where blocked interval has reason "travel_buffer"
    - Contractor name + avatar at the top of the lane (fixed header, does not scroll with time)
    - Lane width: calculated as `(screenWidth - timeAxisWidth) / min(contractorsOnPage, 5)`

    - Pagination controls at bottom (page N of M, prev/next buttons, or dots)

    - "Now" line: painted by CalendarGridPainter at the current time, consistent across all visible lanes

    - The entire day view wrapped in a Column: [contractor name header row, Expanded(vertical scroll area with time axis + lanes)]
