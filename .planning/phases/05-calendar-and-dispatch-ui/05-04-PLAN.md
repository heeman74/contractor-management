    **Create `mobile/lib/features/schedule/presentation/widgets/delay_justification_dialog.dart`:**
    A showDialog-based modal for reporting a delay:
    - Use StatefulBuilder inside AlertDialog to manage local state (reason text, selected ETA date).
    - Title: "Report Delay"
    - Fields:
      1. TextFormField for reason: required, labelText "Reason for delay *", hintText "Describe the reason...", maxLines: 3, validator: empty check
      2. ListTile with date picker: "Select new ETA *" label, trailing calendar icon. On tap: showDatePicker with firstDate = tomorrow, lastDate = +365 days, initialDate = job.scheduledCompletionDate + 1 day (or tomorrow if null).
    - Actions: Cancel (pop with false), Submit (validate both fields, pop with true)
    - barrierDismissible: false (required fields — no accidental dismiss)
    - After confirmed submission:
      1. Build delay entry: `{"type": "delay", "reason": reason, "new_eta": etaDate.toIso8601String().substring(0, 10), "timestamp": DateTime.now().toUtc().toIso8601String(), "user_id": currentUserId}`
      2. Decode existing job.statusHistory JSON, append delay entry, re-encode
      3. Update job in Drift: set statusHistory with new list, set scheduledCompletionDate to new ETA, bump version
      4. Queue sync entry with operation UPDATE for the job (sync queue dual-write)
    - Expose as a static method: `DelayJustificationDialog.show(BuildContext context, JobEntity job, String currentUserId) -> Future<bool>`

    **Update `mobile/lib/features/jobs/presentation/screens/job_detail_screen.dart`:**
    Add a "Report Delay" button on the job detail screen:
    - Visible when job.status is 'scheduled' or 'in_progress' (per CONTEXT.md: "Available on any job in Scheduled or In Progress status")
    - Button placement: in the action buttons area of the job detail screen (alongside existing transition buttons). Use OutlinedButton.icon with Icons.schedule_send or Icons.report_problem and label "Report Delay".
    - On tap: call `DelayJustificationDialog.show(context, job, currentUserId)`
    - After successful delay report, show a confirmation SnackBar: "Delay reported — new ETA: {formatted date}"
    - For contractor view: the button text could say "Report Delay" with a subtitle "Update your ETA" (per CONTEXT.md: contractor view shows prompt like "update status or report a delay")

    **Update `mobile/lib/features/jobs/data/job_dao.dart`:**
    Add a `reportDelay(String jobId, String delayEntryJson, DateTime newEta, int currentVersion)` method:
    1. Read current job from Drift
    2. Decode statusHistory, append new delay entry, re-encode
    3. Transaction: update job row (statusHistory, scheduledCompletionDate, version+1, updatedAt) + insert SyncQueue entry with operation UPDATE and full job payload
    4. This follows the transactional outbox dual-write pattern exactly like other job mutations

    IMPORTANT: status_history is stored as JSON TEXT in Drift (SQLite has no JSONB). Decode with `jsonDecode(job.statusHistory ?? '[]')` to get List<dynamic>, append the new entry Map, then `jsonEncode(list)` to store back. Use `dart:convert`.
    IMPORTANT: The delay dialog does NOT change job.status — the job stays in 'scheduled' or 'in_progress'. Only scheduled_completion_date and status_history are mutated.
    IMPORTANT: Offline-capable: the delay report writes to Drift locally and queues to sync. No HTTP call in the dialog itself.
    IMPORTANT: Multiple delays are allowed per job (unlimited). Each creates a new entry. The latest new_eta overwrites scheduled_completion_date.
